## ClashFX 1.0.21

### Bug Fixes / 问题修复

- **Fixed share-link subscriptions with protocols supported by mihomo** — ClashFX now uses mihomo's own subscription converter when a remote subscription returns share links, so generated configs support more protocols and fields including SSR, TUIC, AnyTLS, Mieru, and more complete VMess/VLESS/Trojan/Hysteria options. This fixes cases where the same subscription worked in ClashX Pro but nodes generated by ClashFX could not connect.
- **Added per-subscription User-Agent setting** — Remote config entries now include an optional User-Agent field. Providers that restrict subscriptions by client type can be updated with a custom UA while existing subscriptions continue to use the default `mihomo/1.19.24` UA when left blank.

---

### 问题修复

- **修复 mihomo 已支持协议的分享链接订阅转换** — 当远程订阅返回分享链接列表时，ClashFX 现在优先使用 mihomo 自带转换器生成配置，支持 SSR、TUIC、AnyTLS、Mieru，以及更完整的 VMess/VLESS/Trojan/Hysteria 字段。修复同一订阅在 ClashX Pro 可用、但 ClashFX 生成节点后无法连接的问题。
- **增加每个订阅独立的 User-Agent 设置** — 托管配置现在可以单独设置 User-Agent。对限制客户端类型的订阅服务，可以填写自定义 UA 获取正确订阅内容；留空时仍使用默认 `mihomo/1.19.24`。
